# Safe Code Modify Skill
## Hook 配置
该技能已配置自动触发hook：
- 当用户输入包含"安全修改代码"、"安全修改"、"安全复现"、"安全修改报错"等关键词时，会自动触发safe-code-modify skill
- 配置文件位于 `settings.json` 中的 `hooks.user-prompt-submit`

你必须严格遵守以下工作流程，不得跳过任何步骤。

## 核心原则
1. **先计划，后执行**：严禁直接询问是否修改代码。
2. **原代码保留**：修改时必须将原代码注释掉，而不是删除。
3. **用户确认**：只有用户明确说"确认"、"执行"等肯定词汇后，才能进行文件写入/编辑操作。

## 工作流程 (Workflow)

### 阶段 1：理解与分析 (Analysis)
当用户提出代码修改/复现/修改报错需求时：
先说明已使用安全代码修改技能，并且你将按照以下步骤进行：
1. 仔细阅读并理解用户意图。
2. 读取相关的代码文件（如果尚未读取）。
3. 形成修改方案。

### 阶段 2：汇报计划 (Plan Presentation)
在这一阶段，**只输出文字描述和伪代码结构，不执行实际文件操作**。

你需要输出以下内容：
1.  **修改概述**：用一句话总结你要做什么。
2.  **具体步骤**：分点列出修改的具体位置和详细原因。
3.  **代码对比预览**：
    - 展示**将被注释掉**的原代码片段。
    - 展示**新增**的代码片段，并详细解释每一行代码的意思。
    - 格式示例：
      ```text（修改的原因）
      [原代码 (将被注释)]:
      // 原代码行 1
      // 原代码行 2

      [新代码]:
      // 原代码行 1 (已注释)
      // 原代码行 2 (已注释)
      新代码行 1（详细解释代码）
      新代码行 2（详细解释代码）
      ```

### 阶段 3：等待确认 (Confirmation)
输出以下确认提示：
```text
请确认是否执行上述修改？
- 确认执行：请回复"确认"、"执行"、"yes"、"同意"、"y"等肯定词汇
- 需要调整：请具体说明需要调整的部分
```

### 阶段 4：执行修改 (Execution)
在收到用户确认后，严格按照阶段2代码对比预览来执行实际的文件操作，不允许有其他新操作：
1. 读取目标文件
2. 将原代码注释掉（比如最前边加上`#`）
3. 在注释下方添加新代码
4. 保存文件

### 阶段 5：验证与总结 (Verification)
1. 检查修改后的文件是否符合预期
2. 输出修改总结
3. 提供测试建议（如果适用）

## 重要提醒
- 严禁在阶段2之前读取或修改任何文件
- 严禁在没有用户确认的情况下执行文件操作
- 原代码必须保留为注释，以便回滚和对比
- 每次修改都要遵循完整的工作流程
