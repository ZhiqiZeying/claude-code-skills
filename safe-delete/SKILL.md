# Safe Delete Skill
## Hook 配置
该技能已配置自动触发hook：
- 当用户输入包含"安全删除"、"删除文件"、"移动到垃圾"、"清理空间"等关键词时，会自动触发safe-delete skill
- 配置文件位于 `settings.json` 中的 `hooks.user-prompt-submit`

你必须严格遵守以下工作流程，不得跳过任何步骤。

## 核心原则
1. **安全第一**：绝不直接删除文件，始终移动到垃圾文件夹
2. **先计划，后执行**：严禁直接询问是否删除文件。
3. **用户确认**：只有用户明确说"确认"、"执行"等肯定词汇后，才能进行文件操作。
4. **日志记录**：记录所有删除操作，方便追溯。

## 工作流程 (Workflow)

### 阶段 1：理解与分析 (Analysis)
当用户提出安全删除/删除文件/清理空间需求时：
先说明已使用安全删除技能，并且你将按照以下步骤进行：
1. 仔细阅读并理解用户要删除的文件位置和内容。
2. 确认目标文件的真实路径。
3. 形成删除方案。

### 阶段 2：汇报计划 (Plan Presentation)
在这一阶段，**只输出文字描述和伪代码结构，不执行实际文件操作**。

你需要输出以下内容：
1.  **删除概述**：用一句话总结你要做什么。
2.  **具体步骤**：分点列出删除的具体位置和详细原因。
3.  **删除预览**：
    - 展示**将被移动**的文件路径。
    - 展示**目标位置**（D:\垃圾 文件夹）。
    - 格式示例：
      ```text（删除的原因）
      [将被移动的文件]:
      C:\Users\Zhiqi\.claude\telemetry\1p_failed_events.json
      
      [移动到]:
      D:\垃圾\telemetry_1p_failed_events_20250416.json
      
      [操作说明]:
      1. 首先检查 D:\垃圾 文件夹是否存在
      2. 如果不存在，创建垃圾文件夹
      3. 将文件移动到垃圾文件夹，并添加时间戳防止覆盖
      4. 记录删除日志
      ```

### 阶段 3：等待确认 (Confirmation)
输出以下确认提示：
```text
请确认是否执行上述删除操作？
- 确认执行：请回复"确认"、"执行"、"yes"、"同意"等肯定词汇
- 需要调整：请具体说明需要调整的部分
```

### 阶段 4：执行删除 (Execution)
在收到用户确认后，执行实际的文件操作：
1. 检查文件是否存在
2. 创建 D:\垃圾 文件夹（如果不存在）
3. 生成带时间戳的新文件名（防止覆盖）
4. 执行文件移动操作
5. 记录删除日志

### 阶段 5：验证与总结 (Verification)
1. 检查文件是否已成功移动
2. 输出删除成功信息
3. 提供垃圾文件夹位置信息

## 重要提醒
- 严禁在阶段2之前读取或修改任何文件
- 严禁在没有用户确认的情况下执行文件操作
- 文件总是移动到 D:\垃圾 文件夹，绝不直接删除
- 每次删除都要遵循完整的工作流程
- 垃圾文件夹中的文件可以在需要时恢复

## 使用示例
用户："帮我清理一下 telemetry 目录"
系统会自动触发 safe-delete 技能，然后：
1. 分析 telemetry 目录下的文件
2. 制定移动到垃圾文件夹的计划
3. 等待用户确认
4. 执行移动操作
5. 记录日志并反馈结果
